Раскрывающийся TopLine с элементами ScreenGlide

  1. Описание

    2. Рекламный модуль состоит из двух компонент: большого и малого баннеров TopLine. Оба TopLine – это свободно растягиваемые flash-ролики разной высоты. При загрузке страницы появляется большой TopLine (при этом контент страницы закрывается), его видно заданное количество секунд, затем он заменяется на малый, который остается на странице сайта до окончания просмотра. Далее, при наведении курсора мыши на малый TopLine, показывается большой TopLine, закрывая контент страницы. При переводе курсора мыши с рекламного модуля, баннер принимает форму малого TopLine.

  2. Подготовка баннера

    3. Подготовьте два flash-ролика, которые можно свободно растягивать по ширине и заглушку .gif или .jpeg, чтобы демонстрировать ее тем посетителям сайта, у которых нет необходимой версии flash-плагина. Высота заглушки должна быть равна высоте малого flash-ролика. Ширина – на ваше усмотрение. Растягиваться рисунок не будет.

    Для того, чтобы зарегистрировать событие клик нужен элемент button (при наведении мыши на такой элемент курсор изменяется). Для того, чтобы перейти по клику – используется функция getURL. Ей передается два параметра: куда переходить – переменная link1 и в каком окне открывать – значение _blank.

    В вашем flash-ролике сделайте элемент button. Для этого элемента напишите следующий скрипт:

    on (release){
	getUrl(_root.link1, _root.target);
}

    Распакуйте пример. Откройте текстовым редактором файл script.js. Внесите изменения только в верхнюю часть скрипта, отмеченную как // ****** Editable block ******.

    // ****** Editable block ******

var gifname = 'mts121_760x60v1.gif';
var smallflashname = 'mts121_760x60v1.swf';
var ar_height_small = 60;
var ar_width_small = '100%';
var bigflashname = 'mts121_760x300v4.swf';
var ar_width_big = '100%';
var ar_height_big = 300;
var ar_valign = 1; //top
var ar_align = 1; //left
var ar_flashver = 6;
var ar_alt_link = '';
var ar_zeropixel = '';

var ar_timeout = 5; // in seconds
	
/************************************/

    Переменные:

    • gifname – имя заглушки (gif). Может загружаться со стороннего сервера. Заглушка показывается, если у посетителя отсутствует необходимая версия flash-плагина. Если вы не указали никакой заглушки, а flash-плагина у посетителя нет, посетитель просто ничего не увидит. Показ при этом все равно посчитается. Заглушка показывается в таблице, в отличие от flash-ролика. Высота рисунка (gif) считается равной высоте flash-ролика, в данном примере – 60 px. Ширина рисунка (gif) не указывается, браузер определяет ее сам;
    • smallflashname – имя малого flash-ролика. Впишите ваше имя файла. Если flash-ролик должен загружаться со стороннего сервера, впишите полный адрес, начинающийся с «http://». В последнем случае, загружать его не придется;
    • ar_height_small – высота малого flash-ролика. Впишите ваше значение в px;
    • ar_width_small – ширина малого flash-ролика. По умолчанию указана 100% (изменять не рекомендуется);
    • bigflashname – имя большого flash-ролика. Впишите ваше имя файла. Если flash-ролик должен загружаться со стороннего сервера, впишите полный адрес, начинающийся с «http://». В последнем случае, загружать его не придется;
    • ar_width_big – ширина большого flash-ролика. По умолчанию указана 100% (изменять не рекомендуется);
    • ar_height_big – высота большого flash-ролика. Впишите ваше значение в px;
    • ar_valign – разворот по вертикали. Впишите 0, если баннер должен разворачиваться вверх, или 1, если баннер должен разворачиваться вниз;
    • ar_align – разворот по горизонтали. Впишите 0, если баннер должен разворачиваться влево, или 1, если баннер должен разворачиваться вправо;
    • ar_flashver – минимальная версия flash-плагина, которая необходима постетилю для просмотра ваших flash-роликов. Проверяйте соответствие flash-плагина и роликов;
    • ar_alt_link – альтернативный клик по заглушке (если не нужен, оставьте пустым);
    • ar_zeropixel – вызов стороннего зеропикселя (если не нужен, оставьте пустым);
    • ar_timeout – время в секундах, сколько будет показываться большой flash-ролик. Практика показывает, что пять секунд – оптимальное время;

    Сохраните script.js, баннер готов к загрузке.

  3. Требования

    4. Имена файлов должны содержать только цифры или буквы английского алфавита, или символ подчеркивания, и не должны содержать русских букв, пробелов, кавычек, тире и специальных символов. Если вы загружаете файлы с другого сервера, впишите полный адрес, начинающийся с «http://».

    Не указывайте в интерфейсе для JavaScript баннеров (в том числе и для этого) русский альтернативный текст.

  4. Загрузка баннера

    5. Добавьте баннер типа generic-extension, или genericposter, или JavaScript. Кликните по загрузить баннер и загрузите script.js и подготовленные flash-ролики.

  5. Проверка работоспособности

    6. Проверьте баннер, в том числе, что он кликается и по первому flash-ролику, и по второму, и по заглушке. Для этого укажите такую версию flash-плагина, которой у вас точно нет, если баннер сделан правильно, вы увидите заглушку.

  6. Ограничения на размер загружаемых файлов в системе AdRiver

    • для обычных графических файлов (в интерфейсе обозначены img width x hight) – 50 Кб;
    • для swf-файлов (в интерфейсе обозначены flash width x hight) – 75 Кб;
    • другие типы файлов – 50 Кб;
    • для многокомпонентных баннеров ограничения проверяются отдельно для каждой составляющей.

    Дополнительное увеличение размеров загружаемых баннеров согласовывается с коммерческим отделом и службой технической поддержки.

  7. Пример

    8. Баннер «Раскрывающийся TopLine c элементами ScreenGlide» на коде generic-extension.
